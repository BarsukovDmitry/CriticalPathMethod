1. Описание функций программы

Функции описаны по порядку их расположения в меню.

 

Файл

  • Открыть, Сохранить, Сохранить как. Используются для сохранения и открытия проектов. Применяется формат файла  .cpm  - собственный формат программы CPM.

  • Импорт. Для построения графика нужно скопировать таблицу, описывающую работы. Значения полей должны быть разделены табуляцией, метки предшествующих работ - запятыми. Просто скопировать такую таблицу можно, например, из MS Excel

  • Экспорт->Сетевой график (bmp). Команда выполняет сохранение изображения сетевого графика проекта в распространённый графический формат bmp.

  • Экспорт->Отчёт по проекту (html). Используется для экспортирования полного отчёта в веб-странице. В отчёте отображены название, автор, описание проекта, сетевой график, все итоги проекта, таблицы работ и событий, описание обозначений.

  • Экспорт->Проект для MS Project (txt). Экспорт проекта с при данном ускорении в MS Project через текстовый файл с разделителями табуляции.

 

Правка

  • Упорядочить по рангу. Используется для упорядочения событий на графике по их рангу расстановки и номеру. Ранг расстановки рассчитывается аналогично обычному рангу, но с функцией минимума. По рангу события выстраиваются по горизонтали. События одного ранга выстраиваются по вертикали в соответствии со своим номером.

 

Вид

  • Оформление. Используется для настройки сетевого графика. Цвет событий используется для закрашивания событий, а цвет выделения - для контура выделенных объектов.

 

Проект

  • Автокоррекция. Данная функция может быть использована для сокращения избыточности сетевого графика: удаление лишних фиктивных работ и конечных событий. Рекомендуется использовать, когда все работы проекта уже построены на графике.

  • Рассчитать. Команда используется для выполнения все расчётов в проекте. Для расчёта ускорения используется значение компонента "Ускорение проекта" на панели инструментов "Объекты".

  • Уведомления при расчёте. Параметр используется при расчёте ускорения проекта для вывода ускоряемых работ на всех этапах ускорения.

 

Объекты

  • Пронумеровать по рангу. Нумерует события в соответствии с рангом: события с большим рангом имеют больший номер.

 

Справка

  • Содержание. Используется для открытия данной справки.

  • О программе. Просмотр информации о программе. Также можно написать письмо автору программы с описанием найденной ошибки (желательно с последовательностью действий, которая приводит к ошибке), рекомендациями, отзывами.
