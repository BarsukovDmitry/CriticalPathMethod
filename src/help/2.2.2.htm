    2.2. Построение графика

    Далее нужно построить сетевой график. Строить сетевой график можно 3-мя способами:

  • автоматический (импорт);

  • полуавтоматический;

  • ручной.

Автоматическое построение (импорт)

 

Программа автоматически построит график на основе импортируемой таблицы работ.

Преимущества:

  • наименьшая сложность;

  • высокая скорость.

Недостатки:

  • невозможность контроля за процессом для изменения графика.

Для построения графика нужно скопировать таблицу, описывающую работы. Значения полей должны быть разделены табуляцией, метки предшествующих работ - запятыми. Просто скопировать такую таблицу можно, например, из MS Excel:

Полуавтоматическое построение.

Представляет собой создание работ на основе других предшествующих работ.

Имеет средние показатели сложности, скорости, гибкости в сравнении с автоматическим и ручным способами.

Построение графика начинается с работ, которым не предшествуют какие-либо другие работы. Для создания работы нужно выбрать в контекстном меню узла "Работы" обозревателя проекта пункт "Создать работу..."

Вводим данные работ в окно "Свойства работы".

   Ручное построение

Данный способ состоит в раздельном создании событий и работ. Сначала создаются события, затем они связываются работами. Для создания как событий, так и работ рекомендуется использовать панель инструментов "Объекты":

Преимущества:

  • наибольшая гибкость.

Недостатки:

  • низкая скорость;

  • более высокая сложность в сравнении с др. способами.
